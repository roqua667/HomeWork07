Command{
1. Преимущества использования паттерна
Паттерн Command Pattern позволяет отделить объект, который вызывает действие, от объекта, который его выполняет.
Преимущества
слабая связность между объектами
возможность хранить команды в истории
возможность реализовать undo/redo
легко добавлять новые команды

2.Как добавить новые команды
Создать новый класс команды.
Реализовать интерфейс ICommand.
Определить методы Execute() и Undo().

3.
вызов идет через команду
команда инкапсулирует действие
можно хранить, передавать и отменять команды.
}



Template Method{
1.Паттерн Template Method Pattern позволяет определить общий алгоритм в базовом классе и делегировать некоторые шаги подклассам.
Преимущества
повторное использование общего кода
единая структура алгоритма
возможность легко расширять систему новыми типами объектов
уменьшение дублирования кода

2.Нужно создать новый класс, который наследуется от Beverage, и реализовать методы:
Brew()
AddCondiments()

3.Hook-метод — это метод, который уже имеет реализацию в базовом классе, но может быть переопределён в подклассе.
}
